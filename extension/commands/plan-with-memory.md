# Plan With Memory

Before planning the feature:
1. Read the constitution or project principles.
2. Read the active feature spec, if present.
3. Read:
   - docs/memory/PROJECT_CONTEXT.md
   - docs/memory/ARCHITECTURE.md
   - docs/memory/DECISIONS.md
   - docs/memory/BUGS.md

Then:
1. Identify constraints, prior decisions, and recurring bug risks.
2. Incorporate them into the plan.
3. Call out conflicts between the requested feature and existing memory.
4. Keep the feature spec clean and task-focused.

Output:
- planning assumptions
- risks
- decisions reused from memory
- recommended implementation approach
