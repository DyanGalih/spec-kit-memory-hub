# Capture

Reflect on completed work and update durable memory only if needed.

Inputs to review:
- active spec / plan / tasks
- final implementation summary
- tests or validation results

Write to memory only when something is:
- reusable
- non-obvious
- likely to prevent future mistakes
- an important technical or product decision

Candidate files:
- docs/memory/DECISIONS.md
- docs/memory/BUGS.md
- docs/memory/WORKLOG.md

Avoid recording routine implementation detail.
